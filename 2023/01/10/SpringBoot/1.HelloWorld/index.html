添加父工程标签

在 Maven 工程中，<parent> 标签用于指定当前工程的父工程信息。通常情况下，一个 Maven 工程可能会有多个子工程，而这些子工程共享一个父工程，通过继承父工程的信息来简化配置。以下这个父标签其实就表示当前项目是一个SpringBoot项目。

使用<parent> 标签可以让子工程继承父工程的信息，例如依赖、插件、资源文件等，从而减少配置的工作量。

建议<parent> 标签作为 pom.xml 文件中你自己写的第一个标签。这样可以让 Maven 在解析 pom.xml 文件时更快地读取到父工程的信息，从而更快地初始化当前工程，然后再解析其他的标签。

引入依赖

这个依赖是 Spring Boot 的 web 启动器，它可以让你的应用支持 web 功能，例如 Spring MVC、Tomcat 容器、Thymeleaf、FreeMarker 等。因为 Spring Boot 是一个轻量级的框架，可以让你快速构建任何类型的应用，而不仅仅是 web 应用，所以我们需要手动引入这个依赖。

创建主程序

Spring Boot 的主程序是用来启动这个web应用的，相当于我们以前的构建服务器并运行。它是一个普通的 Java 类，但它通常使用了 @SpringBootApplication 注解来标注。这个注解包含了三个常用的注解：

  • @SpringBootConfiguration：标注这个类是 Spring Boot 的配置类。
  • @EnableAutoConfiguration：启用自动配置。
  • @ComponentScan：启用组件扫描。

通常情况下，你可以在主程序所在的包及其子包中找到所有的组件，所以我们一般把它放在com.xx[.xx]下面。

SpringApplication.run(MyApplication.class, args) 这一句话的作用是启动应用程序，并运行在 Spring Boot 容器中。

当然，主程序类并不是必须的，你也可以使用maven插件或者Gradle 插件来启动web应用，都是可以的。

简化部署

没错，你没有听错！SpringBoot 还能帮助你更快速有力部署项目。

这个插件是 Spring Boot 的 Maven 插件，用于打包和运行 Spring Boot 应用程序。

它主要提供了以下几个常用的功能：

  • spring-boot:run：用于启动应用程序。
  • spring-boot:repackage：用于重新打包应用程序，以便将所有的依赖打包进应用程序。
  • spring-boot:build-info：用于生成 build-info.properties 文件，其中包含了应用程序的版本、构建时间等信息。

导入这个插件后，你直接在Maven的命令，也就是生命周期那里，依次运行clean和package也可以进行打包操作。
